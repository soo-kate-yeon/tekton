# Screen Templates

These templates provide the foundation for Phase C screen generation in Tekton CLI.

## Template Files

### `page.tsx.template`
Main page component template with:
- shadcn/ui component imports (Button, Card)
- Placeholder structure for content composition
- Accessibility-ready markup
- Comment markers for customization
- Responsive layout with Tailwind utilities

### `layout.tsx.template`
Layout wrapper component with:
- Header/footer structure
- Sticky header with backdrop blur
- Flexible content area
- Optional header/footer props
- Responsive design patterns

### `index.ts.template`
Barrel export pattern:
- Re-exports page and layout components
- Clean import syntax for consumers
- Follows module-based architecture

## Template Variables

These placeholders will be replaced during code generation:

| Variable | Description | Example |
|----------|-------------|---------|
| `{{SCREEN_NAME}}` | Original screen name | "UserProfile" |
| `{{SCREEN_NAME_PASCAL}}` | PascalCase name | "UserProfile" |
| `{{SCREEN_NAME_KEBAB}}` | kebab-case name | "user-profile" |
| `{{SCREEN_TITLE}}` | Human-readable title | "User Profile" |
| `{{SCREEN_DESCRIPTION}}` | Screen description | "Manage user information" |
| `{{SECTION_TITLE}}` | Card/section title | "Profile Information" |
| `{{SECTION_DESCRIPTION}}` | Section description | "View and edit profile" |

## Usage in Phase C

These templates will be used by the `generate` command in Phase C:

```bash
# Future Phase C command (not yet implemented)
tekton generate screen UserProfile --with-layout
```

This will:
1. Read these template files
2. Replace placeholder variables
3. Generate complete screen components
4. Integrate with existing project structure

## Customization

To customize these templates:
1. Edit the template files directly
2. Add new shadcn/ui components as needed
3. Modify the layout structure
4. Update placeholder variables
5. Add additional template files (e.g., `form.tsx.template`)

## Design System Integration

These templates are designed to work with:
- **shadcn/ui** components
- **Tailwind CSS** utility classes
- **OKLCH design tokens** generated by Tekton
- **Next.js** App Router (adaptable to other frameworks)
